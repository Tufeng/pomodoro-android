# Proposal: 多场景管理系统

**Change ID**: add-scene-management  
**状态**: 待审批

## Why（为什么做）

当前 App 只支持一套全局计时配置，用户无法根据不同活动（学习、健身、工作）切换专属节奏设置。这导致：
- 用户每次切换场景都需要手动修改设置，摩擦高
- 无法体现「节律」品牌的个性化核心理念
- 新用户缺少引导，开箱体验差

## What Changes（做什么）

1. 新增「场景」数据模型，包含：名称、工作时长、短休时长、长休时长、循环次数
2. 支持创建、编辑、删除自定义场景
3. 内置 3 个预设场景（不可删除）：
   - 🍅 经典番茄（25分工作 + 5分短休 + 15分长休，4次循环）
   - 📚 深度学习（45分工作 + 15分短休 + 30分长休，2次循环）
   - 💪 健身训练（45分工作 + 10分短休 + 20分长休，3次循环）
4. 支持设置默认场景（App 启动时自动加载）
5. 计时进行中切换场景时，弹窗二次确认，确认后重置计时
6. 计时页显示当前场景名称，点击可快速切换

## Impact（影响）

- **数据层**：新增 Scene 实体，Room 数据库增加 scenes 表，迁移版本 +1
- **UI 层**：新增场景列表页、场景编辑页；计时页增加场景名显示区域
- **业务逻辑**：TimerService 需感知当前 Scene，配置来源从全局 DataStore 改为 Scene 对象
- **现有配置**：首次升级时将现有全局配置自动迁移为「我的默认场景」
