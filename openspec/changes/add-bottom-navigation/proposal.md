# Proposal: 底部导航栏重构

**Change ID**: add-bottom-navigation  
**状态**: 待审批

## Why（为什么做）

当前 App 采用单页 + 右上角设置按钮的结构，随着场景管理功能加入，功能入口增多，现有结构无法承载：
- 场景管理是核心功能，不能藏在设置二级页里
- 用户无法快速感知 App 有哪些主要功能

## What Changes（做什么）

将 App 主结构改为底部导航栏，3 个 Tab：

| Tab | 图标 | 功能 |
|-----|------|------|
| 计时 | ⏱ | 当前主页，倒计时界面 |
| 场景 | 📋 | 场景列表管理入口 |
| 设置 | ⚙️ | 通用设置（声音、通知、关于） |

交互规则：
- 计时进行中切换 Tab，计时继续在后台运行，返回「计时」Tab 时恢复显示
- 计时 Tab 图标在计时进行中显示动态进度环

## Impact（影响）

- **架构**：MainActivity 改为 Scaffold + NavigationBar 结构
- **现有页面**：计时页、设置页保留，场景页为新增
- **导航逻辑**：原有右上角设置按钮移除，功能分散到底部 3 个 Tab
